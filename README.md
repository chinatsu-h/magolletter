1．タイトル
まごレター(高齢者向けシンプルメッセージアプリ）

---

2. 概要
きっかけは祖父母と飲食店を訪れた際、タブレット端末の操作に戸惑う姿を見にしたことでした。便利さを前提として作られた仕組みが、必ずしもすべての人にとって
使いやすいわけではないと感じ、この利便性の乖離を開発側の視点から解消したいと考え、アプリの開発を始めました。
飲食店のタブレットは「選択肢が多い」「アイコンの意味が分かりづらい」「操作ミスが不安」「文字が見えにくい」といった課題がありました。  
そこで本アプリでは、「これを触れば楽しいことが起きる」という成功体験を提供することを目的とし、誰でも直感的に使えるシンプルなUIを実現しました。

6. 工夫した点
祖父母と飲食店を訪れた際、タブレット操作に戸惑う様子を観察し、「なぜ使いづらいのか」を分析しました。  
その結果、「選択肢が多すぎる」「アイコンの意味が直感的に理解できない」「操作ミスへの不安」「文字の視認性が低い」といった課題を見つけました。
これらの課題に対し、本アプリでは「まず成功体験を得てもらうこと」を最優先に設計しました。  
そのため、あえて機能を最小限に絞り、ユーザーが迷わず操作できるシンプルな構成にしています。
また、ボタンやテキストの配置・大きさ・情報量を調整し、直感的に操作できるUIを意識しました。  
特に、「押せば何かが起こる」という安心感を持たせることで、操作への心理的ハードルを下げる工夫を行っています。
このように、単に機能を実装するのではなく、「誰のためのサービスか」「どのような体験を提供するか」を重視して設計しました。

---

7．苦労した点・解決策
本アプリは未経験の状態から開発したため、実装方法についてはAIを活用しながら進めました。しかし、単にコードを動かすのではなく、
一つ一つの処理内容を理解しながら実装することを意識したため、学習と開発を並行する難しさがありました。
また、開発に入る前の企画段階においても、「どこまで機能を持たせるか」の設計に苦労しました。高齢者にとって使いやすいアプリにするためには機能を絞る必要がある一方で、
利便性とのバランスを取ることが難しく、必要な機能を整理しながら取捨選択を行いました。
さらに、開発途中でエラーが発生した際には、原因の特定にも苦労しました。エラーメッセージや公式ドキュメントを確認しながら仮説を立て、修正と検証を繰り返すことで解決しました。
この経験を通して、課題に対して設計段階から考える力と、問題発生時に原因を切り分けて解決する力、そして理解を伴った実装の重要性を学びました。
